> ### OSI 참조 모델?

<br/>

OSI 참조 모델은 다른 시스템 간의 원활한 통신을 위해 ISO에서 제안한 통신 규약이다. OSI 7 계층으로 불리는 이 모델은 네트워크 프로토콜 디자인과 통신을 계층으로 나눠 설명한다.

<br/>

##### 왜 계층을 나누었을까?

표준화를 통해 이질적인 포트 문제나 프로토콜 등으로 인한 문제를 해결하여 비용을 절감하기 위해.

계층별 기능과 통신 과정을 단계별로 나누어서 쉽게 알 수 있고, 특정한 곳에 이상이 생기면 그 단계만 수정할 수 있기 때문에 편리하기 때문.

###### 1) 물리 계층 (Physical Layer)

전기적, 기계적, 기능적인 특성을 이용해서 통신 케이블로 데이터를 전송하는 역할을 한다.

ex. 리피터, 케이블, 허브 등

<br/>

###### 2) 데이터 링크 계층 (Data Link Layer)

물리 계층을 통해 송,수신되는 정보의 오류와 흐름을 관리하여 안전한 정보의 전달을 수행할 수 있도록 도와주는 역할을 한다.

ex. 브릿지, 스위치 등

<br/>

###### 3) 네트워크 계층 (Network Layer)

시스템간의 네트워크 연결 관리 기능과 데이터의 교환 및 중계 기능을 한다.

ex. 라우터, IP 등

<br/>

###### 4) 전송 계층 (Transport Layer)

TCP, UDP 프로토콜을 통해 통신을 활성화 한다. 포트를 열어두고 프로그램들이 전송을 할 수 있도록 제공해준다. 이를 통해 양 끝단의 사용자들이 데이터를 주고 받을 수 있다.

ex. TCP, UDP 등

<br/>

###### 5) 세션 계층 (Session Layer)

송,수신 측 간의 통신을 위한 논리적 연결을 담당한다.

ex. API, Socket 등

<br/>

###### 6) 표현 계층 (Presentation Layer)

데이터 표현에 대한 독립성을 제공하고 암호화하는 역할을 담당한다.

ex. JPEG, MPEG 등

<br/>

###### 7) 응용 계층 (Application Layer)

최종 목적지로 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행한다.

ex. HTTP, FTP, DNS 등
